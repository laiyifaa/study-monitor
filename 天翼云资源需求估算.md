# 天翼云资源需求估算

> 22中暑假网课在线学习平台 — 资源申购依据

## 一、项目概况

- **项目名称**：22中暑假网课在线学习平台
- **场景**：高中暑假网课，学生通过钉钉 H5 微应用观看教学视频并记录学习时长
- **组织**：中国电信股份有限公司温州分公司

## 二、基础参数

| 参数 | 值 |
|------|-----|
| 学生总数 | 1,200 人 |
| 最大同时在线 | 800 人 |
| 学习天数 | 12 天 |
| 每人每天 | 2 节课 |
| 每课时长 | 20 分钟 |
| 视频质量 | 1080p |
| 单视频大小 | **160 MB**（1080p 约 8MB/分钟 × 20 分钟） |

## 三、存储估算

| 计算 | 结果 |
|------|------|
| 20 门课 × 160 MB | **3.2 GB** |

视频文件总量仅 3.2 GB，存储在本地服务器即可。

## 四、流量估算

| 计算 | 结果 |
|------|------|
| 1,200 人 × 2 课/天 × 160 MB × 12 天 | **4,608 GB ≈ 5 TB** |

日均流量约 384 GB。

## 五、峰值带宽估算

| 项目 | 值 |
|------|-----|
| 最大同时在线 | 800 人 |
| 1080p 稳态播放码率 | ~8 Mbps/人 |
| 峰值带宽 | 800 × 8 Mbps = **6.4 Gbps ≈ 6 Gbps** |

说明：视频文件 160MB / 20分钟，编码码率约 8Mbps。浏览器 `<video>` 播放时缓冲期短暂跑满后降速到与播放码率一致，稳态带宽 ≈ 视频码率。

## 六、方案 A：本地服务器 + CDN

### 部署架构

```
学生端(钉钉H5) ──HTTPS──▶ CDN ──HTTPS──▶ 本地服务器 Nginx
                                              ├── FastAPI(后端API)
教师端(钉钉H5) ──HTTPS──▶                   ├── MySQL(数据库)
                                              └── /videos/(视频文件)
```

本地服务器同时承担 API 服务和视频文件存储，CDN 从本地服务器回源并加速分发。视频流不经过 FastAPI，由 Nginx 直接 sendfile 输出。

### 本地服务器条件核查

| 项目 | 要求 | 说明 |
|------|------|------|
| **固定公网 IP** | 必须 | CDN 源站需填写固定 IP，动态 IP 更换后需手动改 CDN 配置 |
| **上行带宽** | ≥ 200 Mbps | CDN 冷启动回源峰值约 250 Mbps，日常回源约 50 Mbps |
| **80/443 端口** | 至少开放一个 | CDN 回源用；如被封可用 8443 等端口回源 |
| **网络稳定性** | 7×24 可用 | 1200 学生依赖此线路，断网 = 教学中断 |
| **硬件配置** | 2 核 4 GB 以上 | API 请求很轻，SSD 更佳 |

### CDN 回源带宽

| 项目 | 值 | 说明 |
|------|-----|------|
| CDN 缓存命中率 | ~90% | 视频文件不变，首次访问后长期缓存 |
| 回源流量 | 总流量 5 TB × 10% ≈ **500 GB** | 首次访问 + 缓存过期回源 |
| 峰值回源带宽 | ~250 Mbps | 集中时段冷启动 |
| 日常回源带宽 | ~50 Mbps | 缓存稳定后 |

## 七、方案 B：天翼云主机 + CDN（备选）

### 部署架构

```
学生端(钉钉H5) ──HTTPS──▶ CDN ──内网──▶ 云主机 Nginx
                                           ├── FastAPI(后端API)
教师端(钉钉H5) ──HTTPS──▶                ├── MySQL(数据库)
                                           └── /videos/(视频文件)
```

与方案 A 架构相同，区别在于服务器从本地换到天翼云主机，CDN 回源走内网不受公网带宽限制。

### 服务器配置

| 参数 | 最低配置 | 推荐配置 | 说明 |
|------|---------|---------|------|
| CPU | 2 核 | **4 核** | 当前 2 核够用；多核提升 Python 并发处理能力 |
| 内存 | 4 GB | **8 GB** | 当前 4G 够用；预留 MySQL 缓冲池 + Redis + 多容器并行 |
| 系统盘 | 50 GB HDD | **100 GB SSD** | SSD 数据库读写性能提升 10 倍 IOPS |
| 带宽 | 5 Mbps | **10 Mbps** | API 请求轻量，CDN 内网回源不受此限制 |
| 操作系统 | Ubuntu 22.04 LTS | Ubuntu 22.04 LTS | Docker 兼容性最佳 |

## 八、方案对比

| | 方案 A：本地服务器 + CDN | 方案 B：天翼云主机 + CDN |
|--|------------------------|------------------------|
| 天翼云资源 | 仅 CDN | CDN + 云主机 |
| 月成本 | 低（仅 CDN 流量费） | 中（CDN + 云主机租金） |
| 运维风险 | 高（断网/断电/IP变更） | 低（SLA 99.95%） |
| 回源方式 | 公网回源（占用本地带宽） | 内网回源（不受带宽限制） |
| 扩展性 | 受本地硬件限制 | 随时升配或水平扩展 |
| 适合阶段 | 试点/开发验证 | 生产环境长期运行 |

## 九、补充说明

1. **CDN 缓存策略**：MP4 视频文件内容不变，缓存 30 天可大幅减少回源流量；首次访问冷启动阶段回源较密集，之后趋于稳定
2. **CDN 带宽弹性**：6 Gbps 为峰值，日常平均约 1~2 Gbps，CDN 按量计费无需预留
3. **HTTPS 证书**：钉钉 H5 微应用强制 HTTPS，Nginx 需配置 SSL 证书（天翼云可申请免费证书）
4. **域名备案**：绑定自定义域名做 HTTPS + CDN 加速需备案
5. **防盗链**：Nginx 层配置 Referer 白名单 + CDN 侧配置防盗链，双重保护视频资源
6. **方案切换**：两个方案代码完全相同，仅 CDN 源站地址不同，可随时切换
